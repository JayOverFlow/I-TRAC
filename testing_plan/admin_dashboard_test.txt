I-TRAC ADMIN DASHBOARD MANUAL TESTING PLAN
=========================================

Project: I-TRAC
Page: Admin Dashboard (admin/admin-dashboard.blade.php)
Description: Central management hub for administrators to view system statistics and user records.

SYSTEM OVERVIEW:
- Summary Cards: Real-time counts for Offices, Departments, Faculty, and Staff.
- User Management Table: Searchable, paginated, and filterable list of all registered users.
- Department Filter: Dynamic dropdown to narrow down table results by specific departments.

---

TEST SCENARIO 1: STATISTICS VALIDATION (SUMMARY CARDS)
------------------------------------------------------
Goal: Ensure the summary cards accurately reflect the database status.
Steps:
1. Log in as an Administrator.
2. Observe the four cards at the top: Offices, Departments, Faculty Members, Staffs.
3. Expected Result: 
   - All cards should display an icon and a clear title.
   - The numbers shown should match the actual records in the database.
   - If a count is 0, it should display "0" rather than being empty.

TEST SCENARIO 2: USER DATA TABLE (DATATABLES CORE)
--------------------------------------------------
Goal: Verify the core functionality of the interactive user table.
- Case A (Data Visibility): Check if columns (TUPT-ID, Names, Email, Role, Dept/Office, Type) are populated.
  Expected: Data should be legible; missing values should show "—" or "N/A" as per implementation.
- Case B (Pagination): Click on page numbers at the bottom (1, 2, Next, Previous).
  Expected: Table should update smoothly without a full page reload.
- Case C (Entries per Page): Change "Results" from 10 to 50.
  Expected: Table should expand to show more records if available.

TEST SCENARIO 3: SEARCH FUNCTIONALITY & HUMAN ERRORS
----------------------------------------------------
Goal: Test the robustness of the search bar.
- Case A (Positive Search): Type a known user's Last Name or TUPT-ID into the search box.
  Expected: The table should filter in real-time to match the query.
- Case B (Zero Results): Type a random string of characters (e.g., "xyz789!!").
  Expected: The table should display a "No records found" message.
- Case C (Empty Search): Clear the search box.
  Expected: All records should reappear immediately.

TEST SCENARIO 4: DEPARTMENT FILTERING
-------------------------------------
Goal: Ensure the custom department filter works correctly.
1. Locate the "Filter by Department" dropdown in the table toolbar.
2. Select a specific Department (e.g., "College of Science").
3. Expected Result: The table should only display users belonging to that department.
4. Select "All Departments".
5. Expected Result: The filter should reset and show everyone again.

TEST SCENARIO 5: RESPONSIVENESS & UI/UX
---------------------------------------
Goal: Verify the dashboard remains usable on different screen sizes.
1. Resize the browser window to a tablet or mobile width.
2. Expected Result:
   - Summary cards should stack vertically or wrap properly.
   - The User Table should become scrollable horizontally (Responsive Table).
   - Sidebar/Navbar should collapse into a "hamburger" menu if applicable.
3. Check Hover Effects: Hover over table rows.
   - Expected: Rows should highlight slightly to improve readability.

TEST SCENARIO 6: NAVIGATION & LAYOUT
------------------------------------
Goal: Ensure the sidebar and header elements are functioning.
1. Click the "Sidebar Toggle" icon.
   Expected: The sidebar should expand/collapse correctly.
2. Click on different links in the sidebar.
   Expected: Navigation should lead to the correct respective pages.
3. Verify the Page Title: Check the browser tab.
   Expected: Title should be "Dashboard | I-TRAC".
