I-TRAC LOGIN PAGE MANUAL TESTING PLAN
====================================

Project: I-TRAC
Page: Login (auth/login.blade.php)
Description: Authentication page for registered users.

SYSTEM OVERVIEW:
- Primary Goal: Authenticate users via Email and Password.
- Error Handling: Displays specific messages for invalid credentials or empty fields.
- Secondary Feature: Link to registration page.
- Potential Feature: Email Verification modal for unverified accounts.

---

TEST SCENARIO 1: HAPPY PATH (SUCCESSFUL LOGIN) (Done)
----------------------------------------------
Goal: Ensure a registered user can log in successfully.
Steps:
1. Navigate to the Login page.
2. Enter a valid, registered email address.
3. Enter the correct password.
4. Click "LOG IN".
Expected Result: User is redirected to the dashboard.

TEST SCENARIO 2: INVALID CREDENTIALS (HUMAN ERROR) 
--------------------------------------------------
Goal: Test error handling for incorrect login details.
- Case A (Wrong Password): Enter a valid email but an incorrect password.
  Expected: Error message "Email or password is invalid" appears; inputs highlighted in red.
- Case B (Unregistered Email): Enter an email that is not in the system.
  Expected: Error message "Email or password is invalid" appears.
- Case C (Case Sensitivity): If passwords are case-sensitive, enter password with different casing.
  Expected: Login fails; proper error message shown.

TEST SCENARIO 3: VALIDATION & MISSING INPUTS
--------------------------------------------
Goal: Test mandatory field enforcement.
- Case A (Empty Fields): Leave both Email and Password blank. Click "LOG IN".
  Expected: Fields are highlighted as invalid; "Email or password is invalid" or specific "required" message appears.
- Case B (Missing Password): Enter email but leave password empty.
  Expected: Password field flags as required/invalid.
- Case C (Invalid Email Format): Enter "user@com" or "user@domain".
  Expected: Browser-level or server-side validation error for invalid email format.

TEST SCENARIO 4: UI/UX & NAVIGATION
-----------------------------------
Goal: Verify links and visual feedback.
1. Click "Register" link at the bottom.
   Expected: User should be directed to the Registration page (Note: Check if href holds the correct route).
2. Look and Feel: Ensure the cover image and logo load correctly.
3. Responsive Check: Resize the browser window to see if the two-column layout (Cover vs Form) adapts gracefully.

TEST SCENARIO 5: EMAIL VERIFICATION MODAL (IF APPLICABLE)
---------------------------------------------------------
Goal: Test the "Email Verification" modal logic (detected in code).
1. Identify if the modal is triggered by an "Unverified" account attempt or a specific button.
2. Click "Send Code" in the modal.
   Expected: Success notification that code was sent; UI updates to show email display.
3. Click "Cancel".
   Expected: Modal closes; user stays on login page.

TEST SCENARIO 6: SECURITY & ACCIDENTAL ACTIONS
----------------------------------------------
Goal: Test robustness against user habits.
1. Double Click: Rapidly click "LOG IN" twice.
   Expected: Button disables or only one request is processed to prevent multiple session starts.
2. Browser "Back" Button: After successful login, click the browser's back button.
   Expected: User should remain logged in and not be able to re-submit the form without refreshing.
3. Enter Key: Type credentials and press the "Enter" key on the keyboard.
   Expected: Form submits automatically without needing to click the button.
