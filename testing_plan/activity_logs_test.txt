================================================================================
MANUAL TESTING PLAN: ADMIN ACTIVITY LOGS PAGE
================================================================================
Page: resources/views/admin/pages/activity-logs.blade.php
Objective: Validate the functionality, data integrity, and error handling of the 
           Activity Logs management table.

--------------------------------------------------------------------------------
SCENARIO 1: INITIAL PAGE LOAD & UI VERIFICATION - (Checked)
--------------------------------------------------------------------------------
1. Navigate to the Activity Logs page.
2. Check if the "Activity Log" header is visible in red text.
3. Verify the table columns: Admin, Action, Summary, Details, Date & Time.
4. Confirm default sorting: The logs should be sorted by "Date & Time" in 
   descending order (newest first).
5. Verify that the "Filter by:" dropdown and Search bar are present.

--------------------------------------------------------------------------------
SCENARIO 2: SEARCH FUNCTIONALITY (Checked)
--------------------------------------------------------------------------------
1. Type a known Admin Username in the search box.
   - Result: Table should only show logs for that admin.
2. Type a specific keyword from the "Summary" or "Details" column.
   - Result: Table should filter to show matching rows.
3. Type a specific date (e.g., "May 14").
   - Result: Table should show logs created on that date.
4. Clear the search box.
   - Result: Table should return to its full list.

--------------------------------------------------------------------------------
SCENARIO 3: ACTION FILTERING (Checked)
--------------------------------------------------------------------------------
1. Click the "Filter by:" dropdown.
2. Select a specific action (e.g., "CREATE" or "UPDATE").
   - Result: Table should only show rows matching that exact action.
3. Change the filter to "All Actions".
   - Result: Table should show all log entries again.
4. Combine Search + Filter:
   - Select a filter (e.g., "DELETE").
   - Type an admin name in the search bar.
   - Result: Table should show "DELETE" actions performed by that specific admin.

--------------------------------------------------------------------------------
SCENARIO 4: PAGINATION & LENGTH MENU (Checked)
--------------------------------------------------------------------------------
1. Change the entries dropdown from 10 to 20 or 50.
   - Result: Table should display the selected number of rows.
2. Click the "Next" arrow at the bottom right.
   - Result: Table should navigate to the second page.
3. Click a specific page number (e.g., "Page 2").
   - Result: UI should update the "Showing page X of Y" text accordingly.
4. Click the "Previous" arrow.
   - Result: Table should return to the previous page.

--------------------------------------------------------------------------------
SCENARIO 5: DATA INTEGRITY & TIMEZONE
--------------------------------------------------------------------------------
1. Perform an action in another part of the system (e.g., update a user or role).
2. Refresh the Activity Logs page.
3. Verify the new log entry:
   - Admin: Your username.
   - Action: The correct action type.
   - Date & Time: Should match your current local time (Asia/Manila timezone).
4. Verify "System" logs: If a log has no admin, it should display "System".

--------------------------------------------------------------------------------
SCENARIO 6: HUMAN ERROR & NEGATIVE TESTING
--------------------------------------------------------------------------------
1. Search for a Non-Existent Term:
   - Type "xyz123random" in the search box.
   - Result: Table should display "No matching records found".
2. Invalid Filter Selection:
   - (If applicable) Try to select a filter that has no data.
   - Result: Table should be empty with the "No matching records" message.
3. Rapid Clicking:
   - Quickly switch between filters or page numbers.
   - Result: UI should remain stable and not "flicker" or crash.
4. Window Resizing:
   - Shrink the browser window to mobile size.
   - Result: The table should become scrollable horizontally without breaking 
     the layout.

--------------------------------------------------------------------------------
END OF TEST PLAN
